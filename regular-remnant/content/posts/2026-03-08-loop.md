The single most useful thing you can do to improve the quality of code generated by Claude is to use an agent to review its plans, and then review the new plan, and then review those plans ... and keep going until the agent no longer finds any "critical" issues with the plan. This is a bit like the idea behind the [Ralph Wiggum Plugin](https://github.com/anthropics/claude-code/blob/main/plugins/ralph-wiggum/README.md), but I run it manually and usually only two or three times. 

It is important, however, to do it no matter how trivial the task. I've discovered that Claude can make critical errors with even the most innocuous requests. While running an endless loop like Ralph Wiggum would probably burn through all my tokens in a day, doing a couple of rounds of manual review like this will save you tokens, because you won't have to then waste time [debugging](https://vibing.kerim.one/posts/2026-02-08-debugging/). 

What agents do you use? It depends on the task. I generally install [Superpowers](https://github.com/obra/superpowers) and [Swift Engineering](https://github.com/johnrogers/claude-swift-engineering) when I'm doing macOS coding. There are also some built into Claude that you can use. I prompt Claude to pick the most suitable ones for the plan. Something like this:

> Dispatch multiple code-reviewer agents to validate the plan's diagnosis and proposed fixes against the actual code, including swift-engineering and superpowers code-review agents.

UPDATE: It seems that Claude has created a new built-in slash command `/simplify` which is designed to be run against recent changes to the code. I might take to running this regularly as well. (Haven't tried it yet.)
